# Task: Part tutor generation (real LLM)

## Role
You are a module implementation engineer.

## Goal
Plug a real LLM (via the configured `LLMProvider`) into PartTutorAgent so
each `PartNote` is generated by the model, but still strictly conforms to
the 10-section template.

## Files to modify
- `stem_learning_agent/agents/part_tutor_agent.py`
- `stem_learning_agent/tools/write_note.py` (allowed if the structure
  needs a parser/validator)
- `stem_learning_agent/prompts/part_tutor.md` (refine but keep the 10
  required sections)
- `tests/test_part_tutor.py` (new)

## Files NOT to modify
- `core/schemas.py`.
- Other agents.

## Public interface
- `PartTutorAgent.run(ctx)` (existing).
- `WriteNoteTool.run(...)` (existing, may add validators).

## Requirements
1. Build the prompt from `prompts/part_tutor.md` plus the part's plan,
   prereqs, formulas, examples, visual plan.
2. Send to `ctx.llm` and parse the response.
3. Validate the returned markdown contains all 10 required section
   headers; if not, regenerate up to N=2 times. After that, persist what
   you have, log unresolved_issues, and let the reviewer flag it.
4. Carry SourceRefs through unchanged.
5. For mock LLM tests, make the mock provider return a 10-section
   skeleton so structural tests still pass.

## Tests
- Mock LLM returns a valid 10-section response → note saved with all
  sections.
- Mock LLM returns a malformed response → regeneration attempted; if
  still malformed, `unresolved_issues` populated.

## Definition of Done
- `pytest` green.
- `pipeline run` on samples/course_001 produces drafts that the reviewer
  passes (or only flags `low/medium`, never `high` for missing
  sections).

## Not allowed
- Removing the 10-section requirement.
- Letting the model invent course content without source refs.

## Completion report
Prompt diff, retry behaviour, test summary.
